## 상세 평가

### 1. 🌾 농업 진단 AI — 총점: 38/40 ⭐ 최고점

**개요**: 개도국 농민이 스마트폰으로 작물 사진을 찍고 음성으로 상황을 설명하면 Gemma 4 E4B가 오프라인(온디바이스)에서 즉시 병해충을 진단하고 치료법을 현지어로 안내.

| 평가 항목 | 점수 | 근거 |
|----------|:---:|------|
| 사회적 임팩트 | 9 | 전 세계 5억 명+ 소농 영향. 작물 손실 연 40%가 병해충 원인. 식량 안보 직결 |
| Gemma 4 차별화 | 10 | E4B 이미지+오디오+140개 언어+온디바이스 = 4종 조합 완전 활용 |
| 실현 가능성 | 9 | PlantVillage 데이터셋 공개, Google AI Studio API 즉시 사용 가능 |
| 데모 가능성 | 10 | 사진 찍고 말하면 즉시 진단 → 심사위원이 30초 안에 체험 가능 |

**Gemma 4 차별화 포인트**:
- **이미지**: 작물 잎 사진 → 병해충 시각 분석 (고해상도 비전 인코더)
- **오디오**: 농민 음성 설명 → 추가 맥락 파악 (E4B 전용 오디오 인코더)
- **온디바이스**: 인터넷 없는 농촌에서도 E4B로 로컬 실행 (개도국 핵심)
- **다국어**: 스와힐리어, 힌디어, 벵골어 등 140개 언어 현지어 응답

### 2. 🏥 다국어 의료 보조 — 총점: 31/40

**개요**: 증상을 언어로 설명하거나 부위 사진을 찍으면 Gemma 4가 초기 진단 보조 정보를 제공.

| 평가 항목 | 점수 | 근거 |

| 사회적 임팩트 | 10 | 의료 사각지대 42억 명. 긴급성 최상 |
| Gemma 4 차별화 | 8 | 멀티모달+다국어 활용. 하지만 의료 특화 성능 불확실 |
| 실현 가능성 | 6 | 의료 데이터 라이선스 문제, 규제 리스크, 오진 책임 문제 |

## 최종 추천: 🌾 농업 진단 AI

### 프로젝트명: **CropDoc** — AI-Powered Crop Disease Diagnostic System

### 핵심 가치 제안
> "인터넷 없는 아프리카 농촌에서, 인도 소농이 힌디어로 말하며 사진을 찍으면 — Gemma 4가 온디바이스로 즉시 진단한다."

### 구체적 구현 방향

#### Phase 1: 핵심 기능 (Day 1-2)

```
입력: 작물 이미지 (카메라) + 음성 설명 (마이크)
처리: Gemma 4 E4B (Google AI Studio API)
출력: 
  - 병해충 진단 결과 (병명, 심각도)
  - 치료 방법 (유기농/화학 처리제)
  - 예방 조치
  - 모두 현지어로 출력
```

**기술 스택**:
```
Frontend: Gradio (HuggingFace Spaces)
Backend: Google AI Studio API (gemma-4-e4b-it)
데이터: PlantVillage (38종 작물, 26가지 병해충, 54,306장)
언어: 초기 5개 (English, Hindi, Swahili, Bengali, Spanish)
```

#### Phase 2: 차별화 기능 (Day 3-4)

```
1. 오프라인 모드 (E4B 온디바이스)
   - 4비트 양자화 E4B → 8GB VRAM/RAM에서 실행
   - 농촌 인터넷 없는 환경 대응

2. 멀티모달 강화
   - 이미지 + 오디오 동시 입력
   - 농민이 사진 찍으며 "잎이 노래지고 있어요" 말하면 더 정확한 진단

3. Agentic Workflow
   - 진단 → 인근 농자재 상점 검색 → 날씨 확인 → 치료 타이밍 제안
   - Function calling으로 외부 데이터 연동

4. 히스토리 추적
   - 128K 컨텍스트로 작물 성장 과정 전체 모니터링
   - 주기적 건강 체크 알림
```

#### Phase 3: 데모 최적화 (Day 5)

